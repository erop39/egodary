# eGOdary

Плагинный движок генерации промптов для Illustrious / Anima / Z-Image Turbo.
Преемник прототипа `eGen 8.6` (монолитный HTML/JS) — то же содержание и правила,
но в виде слоистой Python-архитектуры с системой плагинов.

Полный план разработки и обоснование архитектурных решений: см. документ
`egodary_plan_razrabotki.md`, переданный отдельно. Текущий статус по фазам
плана — в `docs/PROGRESS.md`. История изменений — в `CHANGELOG.md`.

## Установка (dev-режим)

```bash
pip install -e ".[dev]"
```

## Быстрая проверка

```bash
egodary version
egodary plugins list
egodary debug
pytest
```

## Веб-интерфейс

После установки (`pip install -e .`) из корня репозитория:

```bash
egodary serve
```

Откройте в браузере: **http://127.0.0.1:8000/**

Альтернатива без CLI:

```bash
python -m uvicorn egodary.api.main:app --reload --host 127.0.0.1 --port 8000
```

Если видите `{"detail":"Not Found"}` на главной странице — на порту 8000
остался **старый** процесс сервера. Остановите все uvicorn (Ctrl+C в терминале)
и запустите снова одной командой выше. Проверка: http://127.0.0.1:8000/api/health
должен вернуть `{"status":"ok"}`.

## Подготовка к переносу

Для handoff/переноса проекта на новую машину:

```bat
prepare_transfer.bat
```

Скрипт выполнит preflight-проверки и создаст архив в `release/`.
Полная инструкция: `TRANSFER.md`.

## Структура

```
egodary/
├── core/        # модели состояния, реестр тегов, отладочный снимок
├── plugins/     # система плагинов: протоколы, манифест, загрузчик
├── content/     # официальные content-pack плагины "из коробки"
├── cli/         # Typer CLI
plugins_user/    # сюда кладутся личные/сторонние content-паки (drop-in, без установки)
tests/
docs/
```

## Откат на раннюю версию

Проект ведётся в git с момента создания (`git init` в фазе 0). Каждая
завершённая фаза/значимое изменение — отдельный коммит с понятным сообщением
и записью в `CHANGELOG.md`. Чтобы откатиться:

```bash
git log --oneline           # найти нужный коммит
git checkout <hash> -- .    # или git revert / git reset --hard <hash>
```
