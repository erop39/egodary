      Tag Studio

      Поиск и управление runtime тегами: category/subcategory, деактивация, дедупликация, миграция.

      Что делает каждый элемент Tag Studio
      • Search — ищет по label, alias и description.
      • Category — ограничивает операции одной категорией (например, environment.location).
      • Subcategory — фильтрует или нацеливает действие на конкретную подкатегорию.
      • Search (кнопка) — выводит список тегов с meta и статусом.
      • Find duplicates — ищет потенциальные дубликаты в категории.
      • Run migration — приводит runtime-теги к актуальной схеме subcategory.
      • Rollback migration — откатывает миграцию runtime-тегов.
      • + Add tag — открывает модалку создания тега в нужной category/subcategory.

      ТЭГ листер

      Выберите category и подгруппу или используйте Search выше. Клик по тегу в результатах — выбор для Edit/Move/Delete.

      —
      Действия с выбранным тэгом
      Выберите тег в списке выше.

      Delete = soft deactivate (is_active=false). Снимите Active only, чтобы восстановить тег.

      Wildcards

      Загрузите текстовый файл со списком фраз (по одной на строку, маркеры */-/• необязательны) — каждая строка станет отдельным тегом в выбранной категории/подгруппе. Алиас (id) и формат тега для каждой модели генерируются автоматически из текста строки. Чекбоксы управляют включением файла целиком или отдельных строк в реальном времени.
